# Rock Nation - Projeto Prático Integrador

Este projeto é uma página web institucional dedicada ao universo do Rock 'n' Roll, desenvolvida como parte do componente de Gestão de Configuração e Testes de Software da Escola do Futuro (UFG/GO).

## 🎸 Descrição do Projeto
O sistema consiste em uma interface simples, porém moderna, que apresenta a história, gêneros e a essência do Rock. O foco principal não é a complexidade do código, mas sim a aplicação rigorosa de fluxos de versionamento e garantia de qualidade.

## 🛠️ Ferramentas Utilizadas
- Linguagens: HTML5 e CSS3.
- Controle de Versão: Git.
- Hospedagem de Repositório: GitHub.
- Editor de Código: VS Code.
- Documentação: Markdown.

## 📁 Estrutura de Pastas
Conforme as boas práticas de GCS, o projeto está organizado da seguinte forma:
- `/src`: Contém o código-fonte da aplicação (HTML e CSS).
- `/docs`: Documentação técnica (Plano de Testes, Casos de Teste e Relatório de Execução).
- `/tests`: Espaço destinado a registros e evidências de testes.
- `README.md`: Documentação geral do repositório.

## 🚀 Como Executar o Projeto
1. Clone este repositório: 
   `git clone https://github.com/klelsonrodrigues/klelson_rodrigues-trabalho-c23-tdwm-efg.git`
2. Navegue até a pasta do projeto.
3. Abra o arquivo `src/index.html` em qualquer navegador web moderno.

## ⚙️ Práticas de GCS Adotadas
Para este projeto, foram implementadas as seguintes práticas de Gestão de Configuração:
- Controle de Mudanças: Uso de Issues no GitHub para registrar melhorias e correções, garantindo que cada alteração tenha uma motivação registrada.
- Rastreabilidade: Commits vinculados diretamente às Issues, permitindo rastrear qual código resolveu cada demanda.
- Versionamento Paralelo: Uso de Branches  para o desenvolvimento de novas funcionalidades sem afetar a estabilidade da branch `main`.
- Padronização: Mensagens de commit claras e objetivas para facilitar a auditoria do histórico.

## 🧪 Estratégia de Testes
A qualidade do software foi validada através de:
- Planejamento: Criação de um Plano de Testes detalhando escopo e critérios de aceitação.
- Casos de Teste: Definição de 5 cenários principais abrangendo funcionalidade, interface e responsividade.
- Execução Manual: Testes realizados passo a passo no navegador, com registro de resultados obtidos e comparação com os resultados esperados.
- Evidências: Documentação textual dos sucessos e falhas encontrados durante o processo de execução.
