# Plano de Testes - Rock Nation

## 1. Objetivo do Teste
[cite_start]Definir a estratégia para validar as funcionalidades da página institucional, garantindo que o layout seja responsivo e que todos os links de navegação funcionem corretamente[cite: 75, 105].

## 2. Escopo
* [cite_start]**O que será testado:** Navegação do menu superior, funcionamento do botão "Saiba Mais", responsividade em dispositivos móveis e carregamento das imagens de fundo[cite: 76].
* [cite_start]**O que não será testado:** Integração com bancos de dados externos ou APIs de terceiros (não contemplados neste projeto simples)[cite: 30, 76].

## 3. Ambiente de Teste
* [cite_start]**Navegadores:** Google Chrome e Firefox[cite: 77].
* [cite_start]**Dispositivos:** Desktop e emulação de dispositivos móveis via VS Code/Navegador[cite: 34, 77].
* [cite_start]**Sistema Operacional:** Windows/Linux/MacOS[cite: 77].

## 4. Critérios de Aceitação
* [cite_start]O site deve carregar sem erros de console[cite: 78].
* [cite_start]Todos os links âncora (#sobre, #generos) devem levar o usuário à seção correta da página[cite: 78].
* [cite_start]O layout não deve apresentar quebras de elementos em telas menores que 480px[cite: 78].
